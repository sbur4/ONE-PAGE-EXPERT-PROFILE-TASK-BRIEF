# STEP 3 — CV STRUCTURE (EPAM TEMPLATE)
Follow the official EPAM CV section order. Present only verified data.
Do NOT include: photo, date of birth, marital status, nationality.

## 3.1 HEADER
- Full name (large, bold, EPAM Blue)
- Current title / grade level
- Location (city, country only)
- Contact row: Email · LinkedIn URL · GitHub URL · Credly URL
- EPAM tenure (if applicable): "EPAM employee since [year]"

## 3.2 PROFESSIONAL SUMMARY
- 3–5 concise sentences
- Cover: total YoE · core domain · 2–3 strongest technologies · one differentiator
- Cross-reference: Telescope summary ↔ LinkedIn "About" ↔ people.epam.com bio
- Write in third person (EPAM standard): "Serhii is a…"

## 3.3 TECHNICAL SKILLS
Group by category, bulleted, concise:
- Programming Languages
- Frameworks & Libraries
- Cloud & Infrastructure
- Databases
- DevOps & Tools
- Methodologies & Practices
Source priority: Telescope Skills tab → LinkedIn Skills → GitHub top languages

## 3.4 PROFESSIONAL EXPERIENCE
Reverse-chronological. For EACH role:
  Company name | Job Title | [Month Year – Month Year]
  Project/Client (if allowed to disclose)
  Role description: 2–3 bullet points of responsibilities
  Key achievements: quantified where possible ("Reduced build time by 40%")
  Tech stack used: comma-separated list
Cross-reference: people.epam.com ↔ LinkedIn ↔ Telescope Work tab
Flag date discrepancies between sources.

## 3.5 PROJECTS (EPAM PROJECT CARDS)

## 3.9 RECOGNITION & AWARDS

## What NOT to do (EPAM corporate standard)

✗ No neon colors, gradients, or glow effects

✗ No rounded pill badges (use 2px radius max)
✗ No particle animations in document/PDF version
✗ No photos of candidate (EPAM CV standard)
